## 🔄 Flows Implémentés

### **Flow Login Classique**
```
Email + Password 
  → POST /auth/login (body: {email, password})
  → Reçoit {token, user}
  → getCurrentUser() pour récupérer ALL data
  → theme appliqué: setTheme(userData.theme)
  → user stocké: setUser(userData)
  → redirection: /feed
```

### **Flow Register Classique**
```
Form (email, password, firstName, lastName)
  → POST /auth/register
  → Reçoit {token, user}
  → getCurrentUser() pour complet data
  → theme appliqué
  → user stocké
  → redirection: /feed
```

### **Flow OAuth 42**
```
User clique "Connexion 42"
  → Redirection Passport 42
  → User autorise sur intra 42
  → Backend retourne tempToken (JWT avec data 42)
  → Frontend: GET /register/42?tempToken=...
  → User voit form Register42 avec données 42 verrouillées
  ├─ User peut changer avatar
  ├─ User rentre password
  └─ User clique "Inscription"
    → POST /auth/42/confirm (password, confirmPassword, tempToken)
    → Backend extrait data 42 du token (ignore modifs frontend)
    → Crée user avec data authentifiée
    ├─ Si déjà existant (409) → redirection login
    ├─ Sinon, retourne {token, user}
    ├─ Frontend upload avatar custom si changé
    ├─ Theme appliqué
    └─ Redirection /feed
```

### **Flow Upload Avatar**
```
User clique "Changer avatar" → SelectFile
  → FileReader.readAsDataURL() → base64
  → POST /api/uploads/avatar (body: {avatarData: base64})
  → Backend sauvegarde: uploads/avatars/{random}_{timestamp}.jpg
  → Retourne: {avatar: "abc-123_1711234567.jpg"}
  → Update user.avatar = "abc-123_1711234567.jpg"
```

### **Flow Affichage Avatar**
```
Component reçoit user.avatar = "abc-123_1711234567.jpg"
  → Vérifie: isLocalAvatar = !startsWith('http') && !startsWith('data:')
  ├─ Si local → useAvatar("abc-123_...jpg")
  │   → GET /api/uploads/avatars/abc-123_...jpg + JWT
  │   → Retourne blob
  │   → URL.createObjectURL(blob)
  │   → <img src={blobUrl} />
  │
  └─ Si URL externe (42) → affiche directement
      → <img src={user.avatar} /> ✅
```

## 🔒 Sécurité Implémentée

- ✅ **JWT obligatoire** pour tous les uploads/downloads
- ✅ **Validation filename** (pas de `..`, `/`, etc.)
- ✅ **Path traversal prevention** - vérification fs.realpath
- ✅ **Pas d'exposition data 42** - encodé en JWT côté backend
- ✅ **Validation password** minimum 6 caractères
- ✅ **Bcrypt hashing** pour tous les passwords
- ✅ **Status codes propres** (409 pour duplicate, 401 pour auth, 404 404 file not found)
